
DevOps monitoring is divided into these 5 essential types:

## 1. Application Performance Monitoring (APM)

APM focuses entirely on the code and how the software is running.

- **What it tracks:** Code-level bottlenecks, database query times, execution latency, and error rates.
    
- **Why it matters:** It helps developers pinpoint the exact line of code or slow database query causing a lag, rather than guessing.
    
- **Popular Tools:** New Relic, Dynatrace, Datadog, AppDynamics.
    

## 2. Infrastructure Monitoring

This tracks the health and availability of the hardware, cloud resources, and backend environments supporting your apps.

- **What it tracks:** CPU utilization, memory consumption, disk storage, and container health (e.g., Kubernetes pods).
    
- **Why it matters:** Ensures your infrastructure has enough capacity to handle user traffic and alerts you to resource exhaustion before an outage.
    
- **Popular Tools:** Prometheus, Zabbix, AWS CloudWatch, Nagios.
    

## 3. Log & Event Monitoring

This involves aggregating and parsing text files generated by applications, operating systems, and servers.

- **What it tracks:** Error logs, access trails, system state changes, and stack traces.
    
- **Why it matters:** While metrics tell you _when_ something is broken, logs give you the granular historical data to explain _why_ it broke.
    
- **Popular Tools:** ELK Stack (Elasticsearch, Logstash, Kibana), Splunk, Grafana Loki.
    

## 4. User Experience Monitoring (RUM & Synthetics)

This evaluates your system from the perspective of the actual user. It is split into two methods:

- **Real User Monitoring (RUM):** Captures live data from actual user sessions (e.g., frontend page load times across different browsers).
    
- **Synthetic Monitoring:** Uses automated scripts to simulate user behavior (like logging in or checking out) at regular intervals to test system availability.
    
- **Why it matters:** Your servers might look healthy, but the frontend could be completely broken or lagging for the user.
    
- **Popular Tools:** Pingdom, Sentry, Datadog RUM.
    

## 5. CI/CD Pipeline Monitoring

Monitoring shouldn't just happen in production; it must track the software delivery pipeline itself.

- **What it tracks:** Build success/failure rates, deployment duration, and test suite execution time.
    
- **Why it matters:** Optimizes the development workflow by surfacing flaky tests or bottlenecks in the automation pipeline.
    
- **Popular Tools:** Jenkins, GitLab CI plugins, SonarQube.
    

> **Interview Tip:** Wrap up by mentioning **The Three Pillars of Observability**—**M**etrics, **L**ogs, and **T**races (often called **MELT**). Explain that an effective DevOps strategy correlates all five types of monitoring using these three telemetry types to get a unified view of the entire stack.
